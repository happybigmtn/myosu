# Maintenance Playbook

## Goal

Keep docs, outputs, and verification surfaces aligned with current repo reality.

## Inputs

- current plans
- current docs
- current verification commands

## Steps

1. Identify stale claims.
2. Check the live repo state.
3. Rewrite only what is outdated.
4. Verify the rewritten material no longer over-claims.

## Proof

- stale references removed
- docs reflect current code, not aspirations

## Example lane

- updating bootstrap artifacts after code drift
